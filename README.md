# Tsukurou（つくろう）

> **今日、なにつくろう？**

手持ちの食材と気分を入力するだけで、AI が献立セットを提案してくれるアプリ。

---

## 📌 どんなアプリ？

毎日の「今日なに作ろう…」という悩みを、AI への丸投げで解決します。

冷蔵庫にある食材と、今日の気分（ガッツリ / さっぱり / 疲れた…）をポチポチ選ぶだけで、
AI が **主菜・副菜2品・汁物・デザート** の献立セットを提案。
各料理にはレシピ検索リンク（クラシル / YouTube）が付くので、すぐ作り始められます。

### ターゲットユーザー

- 20代前後の社会人（性別問わず）
- 料理に時間や思考コストをかけたくない人
- 一人暮らし〜家族持ちまで幅広く

### 課題と提供価値

| | 内容 |
|---|---|
| **課題** | 「今日なに作ろう」と毎日悩むのが面倒 |
| **価値** | 食材＋気分を入力するだけで、献立セットを AI が提案してくれる |

### 差別化ポイント 🌟

**「手持ちの食材で作れるようアレンジしたレシピ」を AI が提案する**こと。

既存レシピをそのまま検索して渡すのではなく、
「豚バラがないなら鶏ももで代用」「ナンプラーがなければ醤油＋レモンで」のように、
**ユーザーの手持ち食材に合わせた代案** を AI が考えて返します。

### AI のトーン

共感的で、友達のように寄り添うトーンにします。

> 例：「今日疲れたよね。ラクなのいこ、ラクなの。」

## 🧩 機能一覧

### MVP（v1）で作る機能

| # | 機能 | 詳細 |
|---|------|------|
| 1 | 食材入力 | 手入力＋音声入力（Web Speech API ※ブラウザ標準の音声認識機能） |
| 2 | ジャンル選択（タップ式） | 和食 / 洋食 / 中華 / 韓国 / エスニック / なんでも |
| 3 | 気分選択（タップ式） | ガッツリ / さっぱり / ヘルシー / 疲れた / おしゃれしたい |
| 4 | 調理時間選択（タップ式） | 10分以内 / 30分 / じっくり |
| 5 | 自由入力欄 | こだわりたい人向けの追加リクエスト（例：「辛いの食べたい」） |
| 6 | 「献立を提案」ボタン | 入力内容をまとめて AI に送信 |
| 7 | 献立セットの表示 | 主菜・副菜1・副菜2・汁物・デザート の5品を表示 |
| 8 | レシピ表示＋検索リンク | AI がアレンジした**材料（分量付き）と手順**を折りたたみ式でアプリ内に表示。補助として「クラシルで検索」「YouTube で検索」リンクも付ける |
| 9 | 共感的な AI トーン | 提案の冒頭にひとこと共感メッセージを添える |

### v2 以降に検討する機能（今回は作らない）

- 昼 / 夜の切り替え

## 🛠 技術スタック

| 領域 | 技術 | 補足 |
|------|------|------|
| フロントエンド | HTML + CSS + JavaScript | フレームワーク（React 等の開発用土台ツール）は使わず、まずはシンプルに |
| AI | Google Gemini API | 献立の生成を担当 |
| サーバー | Vercel Functions | ブラウザと Gemini の仲介役。API キーをここで管理する |
| デプロイ | Vercel | GitHub と連携して自動公開できるホスティングサービス |
| DB | なし | v1 では保存機能がないため不要 |

### データの流れ

```
[ブラウザ（スマホ / PC）]
   │  ① 食材・気分などを送信（API キーは持たない）
   ▼
[Vercel Functions（api/suggest.js）]
   │  ② サーバー側に保管した API キーを使って Gemini を呼ぶ
   ▼
[Google Gemini API]
   │  ③ 手持ち食材に合わせてアレンジした献立セットを生成
   ▼
[Vercel Functions] → [ブラウザ] ④ 献立を JSON（データ形式の一種）で受け取り、画面に表示
```

### 🔒 セキュリティの要点

**Gemini の API キー（利用パスワードのようなもの）は、絶対にフロントエンド（HTML / JS）に書かない。**
ブラウザに置いたキーは誰でも盗めるため、Vercel の「環境変数」（サーバー側の秘密の置き場所）に保管し、
`api/suggest.js` だけがキーを使う構成にします。

## 📂 ファイル構成（予定）

```
tsukurou/
├── index.html        # 画面の骨組み（入力フォーム・結果表示エリア）
├── style.css         # 見た目（モバイルファースト、ニュートラルなデザイン）
├── app.js            # ブラウザ側の動き（入力の収集、音声入力、API 呼び出し、結果表示）
├── api/
│   └── suggest.js    # Vercel Functions。Gemini API を呼び、献立 JSON を返す
└── README.md         # このファイル（要件と設計）
```

各ファイルの役割：

- **index.html** … 画面に「何を置くか」を決める設計図。フォームや結果エリアの構造
- **style.css** … 「どう見せるか」担当。スマホ縦持ち基準、PC でも崩れないレイアウト
- **app.js** … 「どう動くか」担当。ボタン操作 → `api/suggest.js` への送信 → 結果の描画まで
- **api/suggest.js** … サーバー側の仲介役。プロンプト（AI への指示文）の組み立てと Gemini 呼び出し

---

## 🎨 デザイン方針

- **モバイルファースト**：スマホ縦持ちでの操作を最優先。PC でも崩れない
- **洗練された使いやすい UI**：かわいすぎず、男女問わず使えるトーン
- **ニュートラルな配色**：茶色系の「ほっこり料理アプリ」にありがちな雰囲気は避ける
- 入力はできるだけ **タップで完結**：文字を打つのは食材とこだわり欄だけ

## 🤖 AI への指示（プロンプト）設計メモ

`api/suggest.js` で組み立てるプロンプトに含める要件：

1. ユーザーの入力（食材 / ジャンル / 気分 / 時間 / こだわり）を渡す
2. **手持ち食材を優先して使う**こと、足りない材料は**代用案を出す**ことを指示する
3. 出力は必ず「主菜・副菜1・副菜2・汁物・デザート」の 5 品セット
4. 各料理に「料理名」「簡単な説明」「使う食材」「材料（分量付き）」「つくりかた（手順）」を含める
   （検索リンクへの誘導だけでなく、**アプリ内でそのまま作り始められる**ことが肝）
5. 冒頭に共感的なひとことメッセージを添える（例：「今日疲れたよね、ラクなのいこ」）
6. 画面で扱いやすいよう、決まった形式（JSON）で返させる

※ レシピ検索リンクは AI に作らせず、料理名から `app.js` 側で生成する
　（例：`https://www.kurashiru.com/search?query=料理名`、YouTube も同様の検索 URL）
